Lesson 1: The Basics of the Web and HTML

Where to begin

The worldwide web is a collection of HTML docs. The links between these pages is what makes the web web-like.

Major pieces of the web

You: a person.

Your Computer: probably this.

Web Browser: A program that runs on your computer to display files found on the web.

The Internet: The world's largest computer network. Invented by Al Gore.

Servers: Computers that host the files that make up the web

HTTP - the main protocol of the web

HTML = HyperText Markup Language

It's considered the heart of the web. It's made up of:

Text content: what you see

Markup: what it looks like

References to documents: e.g. images + videos

Links to other pages: Like this!

HTML Markup

HTML markup consists of opening tags, closing tags, and contents. All of that combines into an "element." Here is a link to a video that discusses this in more detail.

Elements can either be in-line or block. Block elements create an "invisible box" around the content that sits inside ofthem.

5 most important take-aways from Lesson 1

  1. What a web page is: A web page is a text document written in a language known as HTML. Our web browsers then read, interpret, and display these documents.
  2. How coding works: Coding involves writing text in a language that a computer can understand. The computer then follows the instructions written by the programmer.
  3. Programmers must write their code using the proper syntax, otherwise the computer will not be able to understand it.
  4. Programmers can’t remember everything: There are too many details and variations to keep everything in your head. It's more important to understand the broad strokes. You can actually google lots of specific things rather than trying to memorize them.
  5. Basic HTML vocabulary: A tag is contained within angled brackets. Most tags have what are known as an opening tag and a closing tag. However, some tags do not need a closing tag. They're known as “void” tags. An HTML element refers to everything within a set of opening and closing tags. An attribute is a property of an HTML element.

Lesson 2: Web Pages, Documents, and Structure

How do websites work?

- You can think of a website much like your house.

- HTML files are the structure. Where your wall is placed. What room is your living room, what room is your kitchen, etc.

- CSS files are the style. What color is your carpeting? What decorations do you have up? What color is the wall?

- Javascript files are your interactive components. Your garage door opener or your television remote. They’re changing some element of your house

Everything is a Box!

Everything on a web page is actually a box. It gets changed using style. When you start to accept that everything is a box, you can begin to visualize this to move items around on the page.Always sketch out the boxes to better understand how the web page is structured. Start with the big boxes and work backward from there. Even really accomplished programmers do this :)

Boxes within boxes within boxes...

HTML uses what some refer to as a "tree-like structure" for its documents. That is to say that elements can be placed inside other elements. Tree-like structures are used in a variety of disciplines to bring some sort of hierarchy to things. When you view an HTML page in a web browser, this tree-like structure manifests itself as a bunch of "nested" boxes. For a better visual example, take a look at this work of art, in which Bob Ross is painting a picture of Rick Ross, who is painting a picture of Ross from Friends, who is painting a picture of Bob Ross and so on and so forth.

When viewing actual HTML code, you can get a sense of this tree-like structure by keeping an eye out for all of the indents on the left side of the screen. The more indented an element is, the more nested it is.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

When we talk about avoiding repetition, we're talking about not having to manually rewrite code over and over and over again. This is important because it helps us avoid errors and make changes to our document more efficiently.

It's pretty easy to overlook something. So, if we decide to make a change to our code, it's easy to forget to make that change in every location that the repeated code appears.

We're also able to save time by not manually rewriting code. Just like Andy, I want each "lesson" on this page to have the same style. Rather than manually apply that style to each lesson, I can give each lesson "div" the same "class" then apply one style to that class. The style will then show up for each lesson.

The Wonders of CSS

Cascading Style Sheets, more commonly known as CSS, allow us to make changes to the style of HTML elements. CSS enables us to control things like font size, color, background color, borders, margins, and much more. When we want to apply the same style to a bunch of related elements, we can give those elements a "class" and then apply style to that class.

Lesson 4: Introduction to "Serious" programming

What is Computer Science?

Computer Science is about solving problems by breaking them down into smaller pieces and then providing a detailed sequence of steps that could be used to solve each piece. Those steps can then be executed by a computer!

Computers vs. Most Machines

Most machines can only do one thing, or maybe just a few things. A toaster, for example, can toast stuff. That's basically it. However, a computer can do anything. It is a universal machine that we can program to do anything.

To program a computer to do something, we must provide it with a precise sequence of steps that it can follow. A computer by itself can't do anything. It has a small group of functions that it can execute. It's up to us to provide it with the exact steps we want it to take, if we want it to do anything cool.

Programming Lanuages

A programming language refers to what we will use to tell a computer what to do. What we'll be using is a language called Python. Python is actually known as an interpreter. This means that when we write a program in Python, it interprets what we wrote and then runs the program in a way that the computer can understand.

Grammar

Just like any language, programming languages have a specific grammar to them. Whereas in English, I could write with incorrect grammar and you'd still probably understand what I'm saying, this is not the case with computers. Computers need us to write programs with exactly correct grammar, otherwise they can't do the cool things we want them to. This is that whole technological empathy thing we keep hearing about.

Python Expressions

Python expressions follow specific grammatical rules. Although they can begin at a high level as a combination of non-terminals, we must get more granular until each piece of the expression is a terminal.

For example: An Expression can be formed by combining an Expression with an Operator followed by an Expression. However, before it can be a legal Python statement, you need to break it down into a combination of terminals such as 1 + 1.

Lesson 5: Variables & Strings

What is a variable?

A variable allows programmers to give names to values. We can then manipulate those variables to produce different values.

What does it mean to assign a value to a variable?

Let's say we have a variable called Jimmy_Variable. If we then write the following code:

Variable = 33

Now whenever we write Variable, it will actually refer to the value "33"

What is the difference between what the "=" means in math versus in programming.

The equal sign does not mean equal. It really is like an arrow. It means "Assignment." Put whatever the right side evaluates to into the name on the left side.

Why would variables be useful?

Variables are useful for a number of reasons. First of all, they can make code more readable by letting us use names that make sense to humans. Additionally, they allow us to store the value of important data. They also make it easier for us to make changes to values.

Lesson 6: Input > Function > Output

What is a function?

A function has 3 pieces:

  1. An input
  2. Some set of things that get done to that input
  3. An output

A function can best be exemplified by the underpants gnomes and their quest to make profits off of stealing underpants. Their input was underpants. Their output was profit. However, they never defined what they needed to do to the underpants to make a profit. Similarly, a function only works if you define what you want done to the input to create an output.

What is the difference between making and using a function?

You make a function by beginning with the word def then typing the name of the function followed by the parameters of the function within parentheses. This is followed by the "body" of the function. In the body, you write what you actually want done to the input parameters. As an example, you may write something for a function called square:

def square(x):
answer = x * x
return answer

When you're ready to use your function, you simply place a value in for the parameter and then bingo bango:

print square(8)
>>> 64

How do do programmers use functions to avoid repetition?

Functions can be reused forever and ever. Although we haven't quite learned how to do it yet, we will eventually be able to use functions to recreate HTML code and not have to manually rewrite it each time.

What happens if a function doesn't have a return statement?

If a function doesn't have a return statement, then the computer won't know what to produce. For example, if you do the following:

def square(x):
answer = x * x

print square(4)
>>> None

You'll get nothing.

Lesson 7: Control Flow and Loops

'If' Statements

'If' statements allow us to specify a range of possible actions to take on an input, which could then change the output. This is useful since it can enable us to make our code do something different based on some comparison that is made. It gives us a lot more flexibility and is a building block for creating more complex programs.

'While' Loops

'While' Loops allow us to write code that does the same thing over and over. This is particularly useful for when you're dealing with a list or string containing many elements. Rather than have to write a ton of code to individually act upon elements, a 'while' loop will let you do the same thing to each element. This is particularly useful for automating this very web page.

Solving Problems

Perhaps the biggest takeaway from Stage 2 is the general approach we have been taught tosolving problems. Solving a programming problem really requires you to break that bigger problem down into smaller, simpler problems. By doing so, you're able to take really complex ideas, and create code that solves them.

Lesson 8: Object Oriented Programming

What is Object Oriented Programming?

Object oriented programming is a style of programming focused on how you organize your code and structure your data. It stands in contrast to other styles such as procedural programming, in which the focus is on writing code that gets you logically from Point A to Point B. Object oriented programming wants to focus on the “objects” we want to manipulate, rather than on the logic that is used to manipulate them.

Why is Object Oriented Programming useful (particularly in large programming projects)?

Object oriented programming is useful because it enables us to create code that can be used and re-used in a variety of situations and by a variety of people. Rather than simply writing code that gets us from Point A to Point B, we can create these little pieces of code that are more generalized. Using the idea of “abstraction” we can strip down the characteristics of Point A and Point B and create code that can just get us from one point to another. This could then be used anytime someone wants to get between two points. When dealing with really large programming projects, being able to reuse code is incredibly useful and time-saving.

Objects, Classes, and Instances

It’s kind of hard to define objects, classes, and instances independently of one another. You can sort of think of a class as a set of characteristics or attributes of things. A class isn’t a real thing itself, just descriptors of what a thing could be. It can include both data and methods. Any “things” that get made and possess those characteristics are objects. When you actually create an individual object with those characteristics, it is an instance of that class and becomes an object. I think. It’s sort of confusing.

Here is an example that tries to explain the differences: Let’s say you want to create a “class” called Cars. Cars have certain characteristics that make them cars. They have 4 wheels, and at least 5 windows. They have a steering wheel, an engine, etc. Anything possessing these characteristics would be an object of the class “Car”. When you actually go and build an object containing these characteristics, you create an instance of that class.

Wikipedia tells me that an object is a particular instance of a class, which I think is what my long-winded example above shows.

Method

A method is a programmed procedure defined as part of a class and included in any object from that class. A class can have more than one method and those methods can be re-used in multiple objects.

Module

A module is a unit of software that is both portable and interoperable. This means that this unit of software can be used in a variety of systems and can function effectively with components of other systems. This is a particularly valuable characteristic of software used in creating large programs since you can break complex coding problems down into smaller, simpler pieces and then use and reuse them when appropriate

Library

In programming, a library is a collection of precompiled sequences of code that can be called upon when writing a program. It allows you to reuse certain sequences quickly and easily rather than re-writing everything manually.

Class Variables

A class variable is a variable defined at the class level rather than at the instance level. Every instance of that class shares the class variable. However, there is only one copy of this class variable. Every object of the class shares this one copy. If a change is made to this copy of the class variable, that change is reflected in every object of the class.

Method Overriding

Method overriding allows a subclass to create a specific implementation of a method that is already provided by one of its higher order classes. The implementation that is provided in the subclass overrides whatever came in the higher class

Let’s get Abstract! (How does OOP connect back to other things like HTML and CSS?)

Object oriented programming is a great way to go about creating smaller pieces of code that can feed into each other and enable you to accomplish way more. For example, we can create objects that act as these little machines that pump out HTML without us having to write it all manually. Additionally, we can grab objects that others have made, plug them into our code, and have way cooler stuff with much less effort.
